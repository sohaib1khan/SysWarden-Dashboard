Capabilities

Everything you need.
Nothing you don't.

Inspired by AWS CloudWatch, Uptime Kuma, and Grafana — but fully self-hosted and modular.

⚡

Static Go Agent

Single binary, no dependencies. Runs on any Linux host regardless of glibc version. Self-registers on first run and reconnects automatically.

🔌

WebSocket Relay

Persistent bidirectional connection. The dashboard can trigger on-demand capability fetches from any agent in real time — logs, processes, exec, files.

🔔

Alert Engine

Threshold-based rules with duration windows. Fire to Gotify, ntfy, Slack, Discord, email, or any webhook when things break.

🧩

Plugin System

Drop a shell script in ./plugins/ and it becomes a live metric. Plugin Store distributes scripts to all agents automatically — no restart needed.

📡

Status Monitors

HTTP, HTTPS, and TCP endpoint health checks with uptime percentage, response time, and visual history bars.

📱

PWA Support

Installable on mobile. Push notifications for alerts. Works offline. Full responsive layout — built as a Progressive Web App from day one.

🐳

Container & VM Aware

Native Docker, Kubernetes, KVM, and Proxmox capability support. The agent understands the full stack running beneath it.

🔐

Security First

bcrypt-hashed agent keys. sys.exec allowlist required. CORS, rate limiting, and request size caps enforced at the backend layer.

💾

Import / Export

Export and import your full configuration as JSON — monitors, alert rules, notification channels. Migrate or backup in one click.

Deployment

Deploy your way.

Seven deployment methods for every environment — from a single server to a fleet managed by Ansible.

Install Script Recommended

Auto-detects arch and glibc version, installs the correct binary, registers a systemd service.

SYSWARDEN_BACKEND_URL=http://<ip>:8000 \ sudo bash scripts/install.sh install

Docker

No Go required. Persist credentials in a named volume for seamless restarts.

docker run -d \ -e SYSWARDEN_BACKEND_URL=http://<ip>:8000 \ -v syswarden-data:/root/.syswarden \ syswarden-agent

Docker Compose

Agent on the same machine as the server — uses the built-in agent profile.

docker compose --profile agent up -d

Build Directly

Requires Go 1.22+ on the target host. Prints credentials once on first run.

cd agent go build -o bin/agent ./cmd/agent SYSWARDEN_BACKEND_URL=http://<ip>:8000 \ ./bin/agent

Cross-compile + SCP

Build for any target from your local machine — no Go needed on the remote host.

CGO_ENABLED=0 GOOS=linux GOARCH=amd64 \ go build -o agent-linux-amd64 ./cmd/agent scp agent-linux-amd64 user@host:/usr/local/bin/

Ansible Fleet

Roll out the agent to many hosts at once with a single playbook run.

ansible-playbook \ -i hosts.ini \ syswarden-agent.yml

Plugins

Any script. Any metric.

If your shell can measure it, SysWarden can monitor it. Drop a script and it becomes a live dashboard metric.

docker_containers.sh bash
#!/bin/bash # PLUGIN_NAME: docker_containers # PLUGIN_VERSION: 1.0.0 # PLUGIN_DESCRIPTION: Running container counts # PLUGIN_INTERVAL: 30 # PLUGIN_OUTPUT_SCHEMA: {"running":"int","total":"int"} RUNNING=$(docker ps -q --filter "status=running" | wc -l) TOTAL=$(docker ps -qa | wc -l) cat <<EOF [ {"name": "running", "value": ${RUNNING}, "unit": "containers"}, {"name": "total", "value": ${TOTAL}, "unit": "containers"} ] EOF

Write a plugin in minutes

Any executable script with manifest header comments becomes a metric. The agent handles scheduling, ingestion, and dashboard rendering automatically.

  • 1 Add manifest comments at the top: PLUGIN_NAME, PLUGIN_INTERVAL, PLUGIN_OUTPUT_SCHEMA
  • 2 Print a JSON array of {"name", "value", "unit"} objects to stdout
  • 3 Drop it into ./plugins/ on the agent host — it appears in the dashboard automatically
  • 4 Or upload to the Plugin Store — it syncs to every connected agent within 60 seconds, no restart needed

Configuration

No config files required.

All agent settings are environment variables. Credentials are persisted automatically after first registration.

Variable Default Description
SYSWARDEN_BACKEND_URL http://localhost:8000 Backend base URL — must point to your server
SYSWARDEN_INTERVAL 10 Metric push interval in seconds
SYSWARDEN_AGENT_ID auto after registration Inject to skip re-registration on new deployments
SYSWARDEN_API_KEY auto after registration Inject to skip re-registration on new deployments
SYSWARDEN_KEY_FILE ~/.syswarden/agent.key Path to persisted credentials file
SYSWARDEN_PLUGINS_DIR ./plugins Directory scanned for local plugin scripts
SYSWARDEN_STATIC auto-detected Force static binary selection (for RHEL / CentOS / AlmaLinux)
